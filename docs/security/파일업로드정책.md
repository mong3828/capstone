## 파일 업로드 제한 정책


## 정책 (*초안)
*현재 설정된 값은 임의값이며, 추후 해당 값은 실제 서비스 운영 및 성능 측정 후 조정의 여지가 있음

### 1) 파일 크기
- **MAX_UPLOAD_BYTES**: 10 MB
- **검증**: 초과 시 413 Payload Too Large(또는 400) + 표준 에러 payload

### 2) CSV 구조 (행/열)
- **MAX_ROWS**: 100,000 (MVP)
- **MAX_COLS**: 200 (MVP)
- **검증**: 초과 시 400 + 원인 코드(예: `LIMIT_EXCEEDED`)

### 3) 처리 타임아웃 (서버)
- **MAX_PROCESS_SECONDS**: 30s 
- **검증**: 초과 시 Job Fail로 기록(추후 구현 예정), 사용자에게는 1주차에서 구현한 일반화된 실패 사유(500) 제공

### 4) 콘텐츠 타입/확장자
- 허용: `text/csv` (+ 필요 시 `application/vnd.ms-excel`)
- 거절: 그 외는 400

---

## 적용 위치(설계)
- **3주차 pre-signed URL 발급 시점**: 파일 메타데이터(파일명/타입/예상 크기)를 기준으로 1차 차단
- **4주차 처리 실행 구현**: 실제 다운로드/파싱 후 행/열/인코딩 검증
