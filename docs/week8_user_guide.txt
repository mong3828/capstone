================================================================================
B2MARK 8주차 — 사용자(직접 수행) 작업 가이드
================================================================================
코드 저장소에 포함된 `watermark mint` 는 RPC·컨트랙트·개인키가 있으면 동작합니다.
환경변수 이름은 `.env.example` 과 동일하게 맞추는 것을 권장합니다.

--------------------------------------------------------------------------------
1. 테스트넷(Sepolia 등) 준비
--------------------------------------------------------------------------------
  • MetaMask(또는 동일한 주소 체계)로 테스트넷 계정 생성
  • Faucet 으로 테스트 ETH 확보 (가스비) — **B2MARK_ADMIN_PRIVATE_KEY** 지갑에 입금
  • RPC URL → `.env` 의 B2MARK_RPC_URL

--------------------------------------------------------------------------------
2. DataAsset 컨트랙트 배포
--------------------------------------------------------------------------------
  • `.env.example` 을 복사해 `.env` 를 만들고 최소한 다음을 채웁니다:
      B2MARK_RPC_URL
      B2MARK_ADMIN_PRIVATE_KEY  (Sepolia ETH 가 있는 지갑 하나로 통일)
  • Hardhat 은 `hardhat.config.js` 에서 `B2MARK_ADMIN_PRIVATE_KEY` 만 배포·가스에 사용합니다.
  • 배포 스크립트 `scripts/deploy_DataAsset.js` 는 **owner·authorizedSigner 를 모두
    배포 지갑 주소(ADMIN 주소)로 동일 설정**합니다. 계정을 여러 개로 쪼개지 않습니다.
  • 배포 명령 (프로젝트 루트 B2MARK):
      npx hardhat compile
      npx hardhat run scripts/deploy_DataAsset.js --network sepolia
  • 출력된 주소를 B2MARK_CONTRACT_ADDRESS 에 기록

--------------------------------------------------------------------------------
3. mint 에 필요한 키 (CLI)
--------------------------------------------------------------------------------
  • B2MARK_ADMIN_PRIVATE_KEY (필수)
      - 컨트랙트의 `authorizedSigner` 와 짝이 맞는 개인키 (배포 시 동일 지갑이면 그대로)
      - structHash 에 EIP-191 서명 생성
  • B2MARK_USER_PRIVATE_KEY (선택)
      - 실제 mint 트랜잭션 가스를 낼 **다른** 지갑을 쓸 때만 설정
      - 비우면 ADMIN 키로 서명·결제 모두 수행
  • 보안: 개인키를 저장소·채팅에 넣지 말 것.

--------------------------------------------------------------------------------
4. CLI 실행 예시
--------------------------------------------------------------------------------
  .env 에 RPC/컨트랙트/ADMIN 키를 넣은 뒤:

     watermark mint -i 결과.csv --metadata-uri ipfs://Qm.../meta.json

  해시만 넣을 때:

     watermark mint --data-hash <64자리hex> --metadata-uri ipfs://...

--------------------------------------------------------------------------------
5. E2E 데모 영상 (8주차 산출물)
--------------------------------------------------------------------------------
  • insert → hash → mint 흐름 녹화
  • 익스플로러에서 tx / tokenId / owner 확인 화면 권장

--------------------------------------------------------------------------------
6. 문제 해결
--------------------------------------------------------------------------------
  • RPC 연결 실패: B2MARK_RPC_URL 확인
  • invalid signature: 배포 시 authorizedSigner 가 ADMIN 주소와 같은지, mint 시
    서명 키가 그 주소와 짝인지 확인
  • 가스 오류: EIP-1559 등은 필요 시 `core/onchain.py` 트랜잭션 빌드 조정
