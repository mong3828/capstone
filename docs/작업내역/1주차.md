# 🚀 1주차 프로젝트 진행 리포트
## 1. 개요 및 목표
이번 주는 프로젝트의 지속 가능성과 보안성을 확보하기 위해 로컬 개발 환경 표준화 및 클라우드(AWS) 보안 인프라 설계에 집중했습니다.

## 2. 주요 수행 작업
### 2.1) 🛠 환경 구축 및 서버 골격 (Backend)
- pyproject.toml (환경 설계도): 모든 팀원이 동일한 Python 버전 및 라이브러리(FastAPI, Pytest 등)를 사용할 수 있도록 표준화했습니다.

- main.py (서버 본체): FastAPI를 활용한 서버 기초 기능을 구현했으며, 향후 확장성을 고려해 요청 ID 기반 로깅 시스템을 구축했습니다.

- test_health.py (자동화 검사): 서버 상태를 시스템이 0.1초 만에 확인하는 자동화 스크립트를 작성하여 품질 검사 시간을 단축했습니다.

### 2.2) 🔐 보안 및 설계
- 보안 문서화: v1.md(위협 모델링)와 acceptance-criteria-v1.md(검증 계획)를 통해 잠재적 공격 루트를 분석하고 방어 전략을 수립했습니다.

- API 명세서: openapi-v1.yaml 작성을 통해 프론트엔드와 백엔드 간의 소통 기준을 마련했습니다.

- AWS 인프라 설계: 루트 계정 MFA, IAM 사용자/역할 분리, S3 버킷(원본/결과 분리) 및 DB 엔티티 설계를 완료했습니다.

### 2.3) 🤖 자동화 파이프라인 (DevOps)
- GitHub Actions (CI): 코드를 올릴 때마다 시스템이 이 자동으로 다음 검사를 수행하도록 설정했습니다:

- Backend Tests: 작성한 테스트 코드 실행

- Bandit: 파이썬 코드 보안 취약점 점검

- Trivy: 오픈소스 라이브러리 및 파일 시스템 보안 스캔
