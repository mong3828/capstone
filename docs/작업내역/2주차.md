# 🚀 2주차 프로젝트 진행 리포트



## 1. 개요 및 목표

이번 주는 안전한 데이터 저장소(DB)를 구축하고, 타인의 데이터를 훔쳐보는 IDOR(Insecure Direct Object Reference) 취약점을 원천 차단하기 위한 철통같은 권한 관리 로직을 백엔드에 반영하는 데 집중했습니다.



## 2. 주요 수행 작업



### 2.1) 🛠 데이터베이스 및 API 구현 (Backend)

- **DB 스키마 및 모델 구현**: 로컬 환경을 위한 SQLite와 SQLModel을 결합하여 `Tenant`, `User`, `Job`, `File`, `AuditLog` 등 핵심 데이터 모델을 코드로 완벽하게 구축했습니다.

- **Jobs API 및 권한 검사**: `/jobs` API의 최소 기능을 구현하고, 타인의 데이터 접근을 차단하는 객체 수준 권한 검사(object-level auth) 함수를 API 로직에 성공적으로 결합했습니다.



### 2.2) 🔐 애플리케이션 보안 정책 (AppSec)

- **권한 모델 명세화**: `authz-model-v1` 문서를 작성하여, 주체와 객체 간의 "허용/거절(403) 규칙"을 API 단위로 명확히 매핑하고 고정했습니다.

- **DoS 방어 기준 설정**: 업로드 제한 정책 문서를 선제적으로 작성하여, 대용량 악성 파일 등을 통한 서버 자원 고갈(DoS) 공격을 방어하기 위한 1차 차단 기준을 확립했습니다.



### 2.3) 🤖 보안 자동화 및 테스트 (DevSecOps)

- **보안 회귀 방지 테스트**: 기존 구축된 CI 파이프라인(Bandit, Trivy, Pytest)의 원활한 흐름을 유지하면서, 권한 없는 접근 시 403 Forbidden 에러가 정상 반환되는지 검증하는 단위 테스트를 추가해 보안 로직의 회귀(Regression)를 방지했습니다.
